## 1. User Intent Discovery

### 핵심 문제
- 생산 현황이 사내 NAS 엑셀에만 존재 → 사외 접근 불가
- 영업사원이 거래처 방문 중 현황 확인하려면 생산관리부에 전화
- IT 매우 낙후된 환경 → 극도로 쉬운 사용성 필수

### 대상 사용자
- 1차: 외근 영업사원 (모바일로 거래처 현황 확인)
- 2차: 대표이사 (AI 도입 효과 체감)

### 성공 기준
1. 영업사원 → 생산관리부 전화 건수 감소
2. 대표이사 만족 ("이게 AI구나" 체감)
3. IT 비숙련자도 즉시 사용 가능한 사용성

## 2. Alternatives Explored

| 방식 | 장점 | 단점 | 판정 |
|------|------|------|------|
| **A. 정적 웹페이지** | 서버 없음, 무료 배포, 단순 | 수동 갱신 (추후 자동화) | **채택** |
| B. 웹앱 + DB (BaaS) | 실시간, 업로드 기능 | 복잡, 유지보수 필요 | 보류 |
| C. 카카오톡 봇 | 앱 설치 불필요 | API 비용, 개발 복잡 | 보류 |

## 4. 기술 설계

### 아키텍처
```
엑셀 파일 → Python 변환기 → data.json → 정적 HTML/JS → Vercel/GitHub Pages
```

### 기술스택
| 구성요소 | 기술 | 이유 |
|----------|------|------|
| 변환기 | Python + openpyxl | 엑셀 파싱 표준 |
| 프론트엔드 | 순수 HTML/CSS/JS | 프레임워크 불필요, 한 파일 |
| 데이터 | JSON | 정적 호스팅 호환 |
| 배포 | Vercel 또는 GitHub Pages | 무료, HTTPS 기본 |

### 데이터 구조 (엑셀 → JSON)
```json
{
  "updated_at": "2026-04-06 09:00",
  "summary": {
    "total_items": 35,
    "total_clients": 11
  },
  "items": [
    {
      "equipment": "12호기",
      "client": "교원턴키",
      "product": "턴키_라이프_교원예움장례식장_브로슈어",
      "data_status": "완료",
      "paper": "240랑데뷰.울화",
      "size": "국전",
      "quantity": 1600,
      "plates": 8,
      "category": "수정",
      "post_process": "선우",
      "deadline": "2026-04-15"
    }
  ]
}
```

### 파일 구조
```
gabwoo/
├── 생산진행현황/          # 원본 엑셀 (건드리지 않음)
│   └── 재단지시서_2026년.xlsx
├── convert.py             # 엑셀 → JSON 변환 스크립트
├── docs/                  # Plan 문서
└── web/                   # 배포할 정적 파일
    ├── index.html         # 모바일 웹앱 (SPA, 한 파일)
    └── data.json          # 변환된 데이터
```
